B. Build the Hackystat Sensor for Eclipse

1. Checkout or download hackyBuild and hackyKernel module as well.

If you have not had the hackyBuild and hackyKernel module in your local environment, you need to checkout these from cvs (consult Philip Johnson for cvs availability), or download the stable version from http://hackydev.ics.hawaii.edu

2. Read README.html in hackyBuild module for general environment setting and build.

There is a detail information on how to set your local environment for hackystat development. Please try to read at least Section A (1: Build Tools installation, 2: Hackystat system configuration, and 3: Hackystat system deployment).

3. Add hackyEclipse.available statment in the hackystat.properties file.

In section 2.3. Configure the hackystat.properties file, you need to add the following statement:

hackyEclipse.available=true
This enables hackyBuild to know hackyEclipse module is available so that eclipse sensor can be build as well in build process.

4. Build Hacky Eclipse Sensor.

To build Hacky Eclipse Sensor, you can type the following in the hackyBuild directory. Note that hackyBuild is the central build module of all hacky related modules so that all ant command would be executed in hackyBuild.

All platforms % ant

After executing the "ant" command, the Hacky Eclipse Sensor would be build inside the hackybuild\build\war\download\eclipse\

C. Hacky Eclipse Sensor Development

You can develop the Hacky Eclipse Sensor easily without deploying modified eclipse sensor in the Eclipse IDE. Before starting the following process, YOU MUST INSTALL ECLIPSE IDE. see the A. Installation.

  1. Import Hacky Eclipse (this module) into Eclipse IDE.
    To import the Hacky Eclipse, You can go to "FIle | Import | Existing Project Into Workspace | Browse your package contents". Check if there is error task (red cross mark icon on the top of project called Hacky Eclipse). If there is, you fix the error or consult me.
  2. Prepare runtime workbench execution for the Hacky Eclipse.
    Even though you have had old (current) eclipse sensor in your Eclipse IDE, you would not be worried about the current eclipse sensor to be run at runtime. you can run the editing (imported) eclipse sensor (the Hacky Eclipse package you opened in Eclipse IDE) from Eclipse IDE. You can go to "Window | Preferences | Plug-In Development | Target Platform", click "Not In Workspace" button to see if only org.hackystat.stdext.sensor is unchecked. This setting gives you not to run current installed eclipse sensor for next eclipse runtime plugin environment so that you can only run your editing (imported) Hacky Eclipse. Don't forget to click "OK" on the right bottom after this change.
  3. Add System.out.println() to see if you can debug a source code.
    To see how it works, Try to add System.out.println("Part was activated!"); into the partActivated() method of the org.hackystat.stdext.sensor.EclipseSensor.PartListenerAdapter class. You can add the statement in the first line after the partActivated method declaration, and save the file.
  4. Run runtime workbench for the Hacky Eclipse.
    To run the new modified Hacky Eclipse sensor, You can go to "Run | Run As...| Run-time Workbench" to open up new Eclipse. After that, Try to click text editor tags on the Eclipse IDE. You can see the "Part was activated!" output in the source Eclipse IDE Console window.
  5. Use debug mode too.
    To use debug functionality, you can double- click the area in the left most tiny bar (before the line number if any, or before the source line begins) in order to set a break point. The light-blue round mark icon would appear at the same source code you added the 3. Add System.out.println().... If you can see the icon, Try to execute run-time workbench with debug mode. Go to "Run | Debug As..| Run-time workbench". You can see that the program stops at the break point in the source Eclipse IDE when you click any text editor tags in the the debug mode in a new opened-up Eclipse IDE.

E. Developer documentation

Developer documentation for the Hacky Eclipse Sensor system is available as java documentation in the hackyBuild\build\reports\javadoc\hackyEclipse . To build javadoc for the Hackystat Eclipse Sensor, type the following in the hackyBuild directory.

Build javadoc % ant javadocAll
